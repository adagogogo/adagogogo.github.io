Design the tables



For efficient future maintenance, the database is structured into three interconnected tables: talks, tags, and talk_tags.

- Talks to TalkTags: One-to-Many. Each row in talks can be associated with multiple rows in talk_tags.
- Tags to TalkTags: One-to-Many. Each tag in tags can be linked to multiple talks in talk_tags.
- Talks to Tags: Many-to-Many. Realized through the talk_tags junction table. A talk can have multiple tags, and a tag can be associated with multiple talks.
