TED Talks Scraping

A Python project combining frontend scraping and API techniques to extract title, speaker, duration, views, and tags from TED Talks webpages.

Why TED Talks?

TED Talks, originating in 1984 as a convergence of Technology, Entertainment, and Design, has grown into a diverse global platform featuring over 6000 talks covering a wide array of topics from science and business to arts and global issues.

My personal motivation for scraping TED Talks data stems from a profound inspiration drawn from numerous talks, fueling a strong interest in exploring and understanding the nuances and impact of these influential presentations.

How many pages to scrape?


The answer is two.
1. One is the main page, the main purpose is to get all urls.
2. The other one is the detail page of each talk. Use the urls scraped from main page, to extract the rest data.

Extract Page 1 - Main Page

Method 1 - From Frontend


What data to extract?
objectID (talk ID), url, duration (video length), title, speakers

Analyze the page
At the moment I started scraping, there were 6271 talks available. However, only 24 talks loaded initially on the first page. Subsequently, clicking the 'Show 24 more' button reveals an additional set of 24 talks with each click, totaling approximately 261 clicks required to view all 6271 talks.

Steps (see the right picture)
1. Loop to click "Show 24 more" button until loading whole talks.
2. Download data including all urls.

Problems
1. Time-Consuming: The process requires over 260 clicks, accompanied by a waiting period of several seconds for each new talk to load.
2. Network Risks: Any network error encountered during the process necessitates restarting from the beginning.

Result
After spending hours on downloading without completion, I decided to switch strategies and use an API instead.

Method 2 - Use API


Analyze the page
1. Upon inspecting the page, it's observed that a POST request is dispatched to a specific URL, carrying the pre-arranged payload.
2. The response, delivered in JSON format, contains all the relevant data.

Steps
1. Function Definition (fetch_all_pages)
2. Data Fetching in a Loop
3. Making the Request
4. Processing Response Data
5. Progress Tracking
6. Return Fetched Data
7. Saving Data to CSV

Result
The approach proved highly effective, enabling the retrieval of data in just a few minutes.

Extract Page 2 - Detail Page

Method - From Frontend


What data to extract?
view, date, context, description, tag

Analyze the page
1. Views: For "18,675 views", extract only the numerical part, without the word "views".
2. Date: For "• October 2023", remove the "•" symbol to isolate the date.
3. The majority of the data elements can be identified using the 'data-testid' attribute.
4. Tag: Tags are identified not by an ID but by the class attribute 'underline'.

Steps
1. Define fetch_additional_data Function
2. Opens an input CSV file extracted from main page for reading and an output file for writing.
3. Iterate over each row in the input CSV file.
4. Output final CSV file including all data.

Result
This process required several hours to extract information for over 6000 talks.

Final Data



Successfully extracted 6268 talks. The next step involves data cleaning.
